# 판교테크노밸리와 위례신도시 비교 분석

판교테크노밸리와 위례신도시의 업무 집적, 토지이용, 대중교통 접근성, 생활권 인구·종사자, 도로망과 버스 접근성을 비교한 공간분석 프로젝트입니다.

## 웹 페이지

- 제출 원본: `판교와위례비교분석.html`
- GitHub Pages: 저장소 최상단에 동일한 파일을 `index.html`로 복사
- 별도 빌드 과정은 필요하지 않습니다.
- Leaflet과 Chart.js를 CDN으로 불러오므로 인터넷 연결이 필요합니다.

## 분석 목적

1. 업무·연구·판매 기능은 실제로 어느 정도 집적되어 있는가?
2. 용도지역 계획과 실제 건축물 이용은 어떻게 다른가?
3. 철도 접근성이 높으면 업무 집적도 함께 높아지는가?
4. 상주인구와 종사자 구성은 두 지역에서 어떻게 다른가?
5. 도로망, 버스정류장 및 역세권 공급에는 어떤 차이가 있는가?

## 주요 결과

| 지표 | 판교테크노밸리 | 위례신도시 |
|---|---:|---:|
| 분석경계 면적 | 87.3 ha | 150.0 ha |
| 업무·연구·판매 분류 레코드 | 79건 | 26건 |
| 건축물대장상 업무시설 연면적 | 182.1만㎡ | 59.7만㎡ |
| 30분 이내 도달 가능 역 | 66개역 | 88개역 |
| 평균 유효 용적률 | 314.0% | 219.6% |
| 토지이용 혼합도(LUM) | 0.415 | 0.767 |

위례는 철도 접근성 측면에서 판교보다 유리하지만 업무·연구·판매 기능의 집적은 판교보다 낮게 나타났습니다. 따라서 업무 집적의 차이는 교통 접근성만으로 설명하기 어렵고, 토지이용 계획의 실현 방식과 건축물 용도 구성을 함께 살펴봐야 합니다.

> 건축물 수치는 물리적으로 중복 제거된 건물 동수가 아니라 건축물대장 용도 분류 레코드 수입니다.

## 분석 범위

- 판교: 판교테크노밸리 도시지원시설용지 중심 87.3 ha
- 위례: 업무·상업 기능과 준주거지역을 포함한 분석경계 150.0 ha
- 위례 상업지역 토지면적 44.2 ha는 전체 분석경계와 구분하여 해석했습니다.

## 전처리 과정

1. 분석경계를 정의하고 면적 계산용 좌표계 EPSG:5179로 변환했습니다.
2. 건축물대장을 좌표화하여 경계와 결합하고 주용도를 재분류했습니다.
3. VWorld 용도지역을 경계로 잘라 용도별 면적과 비율을 계산했습니다.
4. SGIS 집계구와 경계의 교차면적 비율로 인구·가구·사업체·종사자를 추정했습니다.
5. 판교역과 복정역을 출발점으로 시간대별 철도 도달역을 계산했습니다.
6. 도달역 800m 버퍼 합집합과 집계구를 결합해 도달 가능 인구·종사자를 추정했습니다.
7. 역 500m·1km 버퍼와 경계를 교차해 역세권 면적 비율을 계산했습니다.
8. OSM 도로를 경계로 잘라 도로망 밀도와 최근접 IC 직선거리를 계산했습니다.
9. 버스정류장 중복 위치를 정리하고 경계 내부 정류장 수와 밀도를 계산했습니다.
10. 결과를 GeoJSON·JSON으로 저장하고 최종 HTML에 반영했습니다.

## 산출 기준

- 면적 및 거리 계산 좌표계: EPSG:5179
- 인구·종사자: 집계구 교차면적 비율을 이용한 면적가중 추정
- 도달역: 환승 노선의 동일 물리적 역 통합
- 도달 생활권: 도달역별 직선거리 800m 버퍼 합집합
- 역세권: 운영역의 500m·1km 직선거리 버퍼
- 도로망 밀도: 경계 내부 도로 길이 ÷ 분석경계 면적
- 토지이용 혼합도: 고정 용도 범주의 정규화 Shannon entropy
- 유효 용적률: 0% 초과 2,000% 미만 레코드

## 분석 한계

- 집계구 통계는 건물별 실제 값이 아닌 교차면적 비례 추정치입니다.
- 철도 접근성은 사용한 네트워크와 환승시간 가정에 영향을 받습니다.
- 역세권·버스 생활권은 직선거리 버퍼로 실제 보행경로와 다를 수 있습니다.
- 버스 지표에는 노선 수, 운행 빈도, 혼잡도와 환승 편의가 반영되지 않았습니다.
- OSM 자료는 조회 시점의 편집 상태에 영향을 받습니다.
- 건축물대장 분류 레코드는 물리적 건물 동수와 같지 않습니다.
- 두 분석경계의 계획 목적과 토지 구성이 완전히 같지 않으므로 결과를 단순 인과관계로 해석해서는 안 됩니다.
